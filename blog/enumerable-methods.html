1.4:Enumerable#map

This week we'll be diving a little bit more into one of the methods in Ruby from the Enumerable module.I'm going to keep it fairly simple and hopefully you won't require a .map(first and last pun I promise). The method we'll be looking at is the map method (thus the terrible joke I made).So what does the map method do exactly?
The map method takes an enumerable object(an Array or a Range) and iterates through each element passing a block once. It then returns a NEW Array(this means it's a non-destructive method because it returns a new Array). Before looking at an example of how it works exactly we can also apply a destructive method(modify the original array) by adding the ! symbol after map eg:
array.map!{|i| p i}
By doing this it doesn't return a new Array it affects the original and returns it. What is the difference between the each method and the map method?
The each method will take a block and iterate through the array and do something with the block, but it still returns the original array. The map method will take a block, iterate through the array and return a new array.Here's an example:
y = [1,2,3].each {|x| x + 2} will still return [1,2,3]
whereas
x=[1,2,3].map {|x| x+2}
will iterate through the array and pass the block returning a new array with as [2, 3, 4]. Another way of wording the map method is using the term collect. They both do the exact same thing. Hopefuly this wasn't too complicated to understand! See you next week!
